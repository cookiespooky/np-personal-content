# Notepub — статический генератор сайтов из Markdown без CMS

Если вам нужен быстрый, SEO-дружелюбный сайт без админки и базы данных, Notepub может быть альтернативой WordPress, Ghost и другим CMS.

Notepub — это статический генератор сайтов, который создаёт чистый HTML из Markdown и декларативных правил. Он подходит для блогов, документации, SEO-проектов и контентных сайтов.

---

## Что такое Notepub

Notepub — это статический генератор сайтов (static site generator), который:

- генерирует чистый HTML без JavaScript-фреймворков
- работает без CMS и базы данных
- использует Markdown как основной формат контента
- поддерживает декларативную модель через `rules.yaml`
- подходит для Git-workflow и автоматического деплоя

В результате вы получаете:

- быстрые страницы
- предсказуемую структуру
- минимальный технический долг
- хорошую индексируемость поисковыми системами и ИИ

---

## Почему сайт без CMS может быть лучше

Большинство сайтов строятся на CMS: WordPress, Joomla, Bitrix и других. Это удобно, но создаёт:

- лишнюю сложность
- зависимость от плагинов
- проблемы со скоростью
- уязвимости

Статический сайт из Markdown решает эти проблемы:

- нет админки — нечему ломаться
- нет базы данных — нечему падать
- нет лишнего JavaScript — высокая скорость загрузки
- хостинг возможен на GitHub Pages или любом CDN

Notepub делает этот подход системным и масштабируемым.

---

## SEO для статических сайтов

Одно из главных преимуществ статического генератора — предсказуемая SEO-архитектура.

Notepub генерирует:

- чистые URL
- корректную иерархию заголовков
- sitemap.xml
- robots.txt
- JSON-LD (структурированные данные)
- внутренние ссылки и хабы

Поскольку сайт — это готовый HTML, поисковые системы индексируют его без проблем рендеринга.

Это особенно важно для:

- SEO-блогов
- контентных проектов с тысячами страниц
- документации
- тематических хабов

## Сайт из Markdown: как это работает

Контент пишется в Markdown:

---

title: Пример статьи
type: article
tags: [seo, static-site]

---

Текст статьи…

Далее Notepub:

1. Читает структуру контента
2. Применяет правила из `rules.yaml`
3. Формирует страницы, хабы, коллекции
4. Генерирует статический HTML

В результате сайт становится описательной моделью, а не набором вручную настроенных шаблонов.
