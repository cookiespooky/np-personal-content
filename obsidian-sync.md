# Синхронизация Obsidian между устройствами - полный обзор всех способов

Obsidian - локальный инструмент. Файлы лежат у вас на диске в виде обычных `.md`. В этой статье - все реальные способы синхронизации, их логика, плюсы, минусы и уровень сложности.

---
## 1. Официальный Obsidian Sync

Платный сервис от разработчиков.

## Плюсы

✔ Работает на всех платформах  
✔ Хорошо обрабатывает конфликты  
✔ Шифрование  
✔ Версии файлов  

## Минусы

✖ Платный  
✖ Закрытая инфраструктура
✖ Не оплачивается Российскими картами

**Кому подходит:** если не хочется думать о синхронизации вообще.

---

## 2. iCloud (нативный способ в экосистеме Apple)

**Тип:** облачная синхронизация
**Для кого:** простой бесплатный вариант без глубоких технических знаний

Если вы используете:

- Mac
- iPhone
- iPad

то самый простой способ синхронизации - **iCloud Drive**.

Obsidian на macOS и iOS умеет работать напрямую с iCloud-папкой.  
Никаких плагинов не требуется.

### Как это работает

1. Vault создаётся внутри папки iCloud Drive.
2. iCloud автоматически синхронизирует файлы между устройствами.
3. Obsidian открывает ту же папку на каждом устройстве.

### Плюсы

✔ Максимально просто  
✔ Не требует плагинов  
✔ Работает стабильно внутри Apple-экосистемы  
✔ Нет дополнительной настройки  

### Минусы

✖ Привязка к Apple  
✖ Android не поддерживается нативно  
✖ Нет продвинутого контроля версий  

### Важно понимать

Obsidian не "синхронизируется через iCloud".  
Он просто работает с папкой, которую синхронизирует операционная система.

Это классическая файловая синхронизация.

## 3. Yandex WebDAV (через Remotely Save)

**Тип:** протокольная синхронизация (WebDAV)  
**Для кого:** простой бесплатный вариант без глубоких технических знаний

Obsidian подключается к Яндекс Диску через WebDAV с помощью плагина Remotely Save.

### Плюсы

✔ Бесплатно  
✔ Работает на всех платформах  
✔ Не требует установки клиента Яндекс Диска

### Минусы

✖ Возможны конфликты при одновременном редактировании  
✖ Медленнее, чем P2P  
✖ Нет продвинутого контроля версий

👉 Подробная инструкция: [[yd obsidian sync|Синхронизация Obsidian через Яндекс]]

---

## 2. AWS S3 (через Remotely Save)

**Тип:** протокольная синхронизация (S3 API)  
**Для кого:** тем, кто хочет более стабильное и масштабируемое решение

Obsidian подключается к S3-хранилищу через плагин Remotely Save.

Можно использовать:

- AWS S3
- Yandex Object Storage
- любой совместимый S3-провайдер

### Плюсы

✔ Более стабильная работа с большими vault  
✔ Можно включить версионирование  
✔ Масштабируемость

### Минусы

✖ Требует понимания access key / secret key  
✖ Платное хранилище  
✖ Более сложная настройка

S3 — это уже шаг в сторону инфраструктурного подхода.

---

## 3. Git (через Obsidian Git Plugin)

**Тип:** репликация через систему контроля версий  
**Для кого:** разработчики и технические пользователи

Vault становится git-репозиторием.

Obsidian Git plugin автоматически делает commit и push.

Можно использовать:

- GitHub
- GitLab
- Gitea

### Плюсы

✔ Полная история изменений  
✔ Чёткий контроль версий  
✔ Подходит для текстов

### Минусы

✖ Конфликты требуют понимания Git  
✖ Не самый простой вход  
✖ Нужно настраивать доступы

Git — самый прозрачный способ, но требует дисциплины.

---

## 4. P2P-синхронизация (Syncthing)

**Тип:** прямое устройство → устройство  
**Для кого:** тем, кто не хочет облако

Устройства синхронизируют папку напрямую.

### Плюсы

✔ Бесплатно  
✔ Без централизованного сервера  
✔ Очень быстро  
✔ Минимум посредников

### Минусы

✖ Устройства должны быть онлайн  
✖ Для удалённого доступа нужен VPN

P2P — один из самых надёжных способов, если устройства часто работают одновременно.

## 7. Self-hosted сервер (MinIO, Nextcloud)

Можно поднять свой сервер:

- S3 (MinIO)
- WebDAV
- Nextcloud

### Плюсы

✔ Полный контроль  
✔ Можно строить свою инфраструктуру  
✔ Подходит для команд  

### Минусы

✖ Требует DevOps  
✖ Нужно обслуживать  

---

## 8. Ручные способы

- Копирование файлов  
- ZIP-архивы  
- Флешка  

Работают, но не масштабируются.

---

## Архитектурное различие способов

Все методы делятся на 4 типа:

### 1️⃣ Файловая синхронизация
Dropbox, Google Drive, iCloud.

### 2️⃣ Протоколная синхронизация
WebDAV, S3.

### 3️⃣ Репликация через VCS
Git.

### 4️⃣ P2P
Syncthing.

Понимание этого помогает выбрать подход.
